BGP: El Border Gateway Protocol (BGP) es un protocolo escalable de dynamic routing usado en la Internet por grupos de enrutadores para compartir información de enrutamiento. BGP usa parámetros de ruta o atributos para definir políticas de enrutamiento y crear un entorno de enrutamiento estable.
  • Router bgp (numero asignado de router)
  • neighbor (direccion ip de nuestro vecino) remote-as (numero asignado de nuestro router vecino)
  • neighbor (direccion ip de nuestro vecino) remote-as (numero asignado de nuestro router vecino)
  • network (direccion ip de nuestro segmento) mask (mascara de red)

telmex(config)# router bgp 10

telmex(config-router)#neighbor 11.11.11.2 remote-as 20

telmex(config-router)# neighbor 14.14.14.1 remote-as 40

NAT La traducción de direcciones de red, también llamado enmascaramiento de IP o NAT (del inglés Network Address Translation), es un mecanismo utilizado por routers IP para cambiar paquetes entre dos redes que asignan mutuamente direcciones incompatibles. Para esta practica NAT (PAT sobrecargado) se establecio la configuración de NAT, y los dominios inside y outside en las interfaces corresponidentes , la regla de nat aplica cuando el paquete arriba en un dominio y sale en el otro, es la regla básica para que la traducción ocurra, y se mantiene una tabla de traducciones de acuerdo a la regla que utilices, como por ejemplo una acces-list.

Etherchannel: Etherchannel de 2 Gbps entre switches. la tecnología EtherChannel es una técnica switch a switch LAN para agrupar varios puertos Fast Ethernet o Gigabit Ethernet en un único canal lógico. Cuando se configura un EtherChannel, la interfaz virtual resultante se denomina “canal de puertos”. En este caso configuraremos LACP, es muy similar a PAgP ya que también puede agrupar puertos con características similares. Es un protocolo definido en el estándar 802.3ad. Los modos de configuración de LACP son:

  • Activo. Está habilitado para iniciar negociaciones con otros puertos.
  • Passive. No puede iniciar negociaciones, pero si responde a las negociaciones generadas por otros puertos.

Dos puertos pasivos tampoco podrán nunca formar grupo. Es necesario que al menos uno de los dos puertos sea activos. En el modo manual toda la configuración del puerto se realiza de forma manual, no existe ningún tipo de negociación entre los puerto

  • channel-group (numero de canal) mode (mode)
  • LACP: channel-group (numero de canal) mode (active)
  • PAgP: channel-group (numero de canal) mode (passive)

VTP server-Troncales 802.1d -VLANS: VTP son las siglas de VLAN Trunking Protocol, un protocolo de mensajes de nivel 2 usado para configurar y administrar VLANs en equipos Cisco. Permite centralizar y simplificar la administración en un domino de VLANs, pudiendo crear, borrar y renombrar las mismas, reduciendo así la necesidad de configurar la misma VLAN en todos los nodos. El protocolo VTP nace como una herramienta de administración para redes de cierto tamaño, donde la gestión manual se vuelve inabordable. VTP opera en 3 modos distintos:

  • server: el equipo es el servidor de VTP.
  • client: el equipo es el cliente de VTP.
  • transparent: no interactua ni con clientes ni servidor

Troncales 802.1d Un enlace troncal es un enlace punto a punto, entre dos dispositivos de red, que transporta más de una VLAN. Un enlace troncal de VLAN le permite extender las VLAN a través de toda una red. Cisco admite IEEE 802.1Q para la coordinación de enlaces troncales en interfaces Fast Ethernet y Gigabit Ethernet. Más adelante en esta sección, aprenderá acerca de 802.1Q

VLANs Las VLAN (redes de área local virtuales) pueden considerarse como dominios de difusión lógica. Una VLAN divide los grupos de usuarios de la red de una red física real en segmentos de redes lógicas

SVI, DHCP, HSRP y STP:
SVI La SVI o Switch Virtual Interface es una interface virtual, como su nombre lo dice. Funciona como una interfaz Capa 2 y Capa 3 y generalmente son la puerta de enlace para los usuarios que pertenecen a este dominio de broadcast (vlan).
DHCP El Servidor DHCP, de sus siglas en ingles Dynamic Host configuration Protocol, es un servidor de Red el cual permite una asignación automática de direcciones IP, getaways predeterminadas, así como otros parámetros de red que necesiten los clientes. El sistema DHCP envía automáticamente todos los parámetros para que los clientes se comuniquen sin problema dentro de la red.
HSRP El Hot Standby Router Protocol es un protocolo propiedad de CISCO que permite el despliegue de enrutadores redundantes tolerantes de fallos en una red. Este protocolo evita la existencia de puntos de fallo únicos en la red mediante técnicas de redundancia y comprobación del estado de los routers.
STP El protocolo Spanning Tree (STP) deshabilita los enlaces redundantes, quedando así una red libre de loops. En caso de que alguno de los enlaces principales falle, uno de los enlaces secundarios será habilitado. Configuramos el modo STP en el que trabajarán los switches.

Troncales Con este comando, la interfaz cambia al modo de enlace troncal permanente. El puerto establece una negociación de protocolo de enlace troncal dinámico (DTP) para convertir el enlace en un enlace troncal.

  • interface (Puerto)
  • switchport mode access
  • switchport mode trunk

s1(config)# interface range g0/1-2

s1(config-if)# switchport mode access

s1(config-if)# switchport mode trunk

VTP Aqui activaremos el protocolo VTP, pero en este caso será en modo cliente ya que esto permite recibir las vlans que se hayan creado en el switch con vto server.

  • vtp domain (nombre de dominio)
  • vtp password (contraseña)
  • vtp mode client

s1(config)# vtp domain iric

s1(config)# vtp password 1001

s1(config)# vtp mode client

Asignacion de VLANS a puertos Después de crear una VLAN, el siguiente paso es asignar puertos a la VLAN. Un puerto de acceso puede pertenecer a una sola VLAN por vez mediante este comando, se determina cuales serán los puertos para conectarse a cierta vlan.

  • interface (range) (Puerto/puertos)
  • switchport mode access
  • switchport access vlan (VLAN)

s1(config)# interface range g0/1-5

s1(config-if)# switchport mode access

s1(config-if)# switchport access vlan 10

s1(config)# interface range g0/6-10

s1(config-if)# switchport mode access

s1(config-if)# switchport access vlan 20

s1(config)# interface range g0/11-15

s1(config-if)# switchport mode access

s1(config-if)# switchport access vlan 30

Comandos de estatus y pruebas

Descripción Sintaxis

Estatus de vtp Conocer el estatus de vtp, es decir, si esta activado o no, y como es que se configuro.

s1# show vtp status

Estatus de stp Conocer el estatus de stp, es decir, si esta activado o no, y como es que se configuro, ademas de conocer quienes son los bridges que estan configurados y quienes no y saber sobre el root.

s1# show spanning-tree

Estatus BGP Conocer de bgo, quienes son sus vecinos, quienes no son sus vecinos y como estan sus rutas.

R1# show ip bgp neighbors

Tabla de ruteo Conocer la tabla de enrutamiento, es decir, saber a que direcciones concoemos y los protocolos con los que se configuraron

R1# show ip route

Vlans y HSRP Conocer la relacion de vlans con sus respectivos HSRP, ademas de conocer el estatus de HSRP y el estatus de las vlans

R1# show standby

Ver las acces-list Conocer que acces-list estan configuradas y que estan permitiendo o denegando

R1# show access-list

ping Saber si podemos establecer una conexion con una direccion ip y si estamos entablando una comunicacion efectiva.

C/usuario_> ping ip(X.X.X.X)

C/usuario_> ping -t ip(X.X.X.X)

Tracert Conocer los salto que hace un ping para poder llegar a la direccion ip de destino

C/usuario_> tracert ip(X.X.X.X)
